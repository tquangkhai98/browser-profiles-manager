                    ✨ NEW — The missing link between AI agents and browser sessions
                

                
AI Agent Integration

                
Login once. Automate forever.

                

                    Bridge the gap between your persistent browser profiles and modern AI orchestration.
                    BPM allows your agents to bypass 2FA and login barriers by leveraging existing
                    authenticated sessions through Model Context Protocol (MCP) endpoints.
                

            


            
            

                
                

                    

                        
🎭

                        

                            
Playwright MCP

                            RECOMMENDED
                        

                    

                    

                        Stable, official Anthropic integration. Uses accessibility tree for reliable element
                        interaction. Best for E2E testing and form automation.
                    

                    

                        
{
  "mcpServers": {
    "browser-admin": {
      "command": "npx",
      "args": [
        "@playwright/mcp@latest",
        "--user-data-dir",
        "~/.local/share/bpm/profiles/lms-admin"
      ]
    }
  }
}

                        
                    

                


                
                

                    

                        
🔧

                        

                            
Chrome DevTools MCP

                            ADVANCED
                        

                    

                    

                        Connect to a running Chrome instance with remote debugging. DOM inspection,
                        Network monitoring, and Console access in real-time.
                    

                    

                        
bpm use lms-admin --debug-port=9222

                        
                    

                    

                        
{
  "mcpServers": {
    "chrome-devtools": {
      "command": "npx",
      "args": [
        "chrome-devtools-mcp@latest",
        "--browserUrl=http://127.0.0.1:9222"
      ]
    }
  }
}

                        
                    

                


                
                

                    

                        
🤖

                        

                            
Browser Use MCP

                            AI-NATIVE
                        

                    

                    

                        AI-native browser automation — describe actions in plain language instead of
                        CSS selectors. The most seamless integration for AI workflows.
                    

                    

                        
{
  "mcpServers": {
    "browser-use": {
      "command": "npx",
      "args": [
        "browser-use-mcp@latest",
        "--user-data-dir",
        "~/.local/share/bpm/profiles/lms-admin"
      ]
    }
  }
}

Use Cases

                

                    

                        
🎓

                        
Multi-Role Testing

                        
Create profiles for Admin, Teacher, Student — login once per role, then let AI test all flows with correct permissions.

                        

                            bpm create lms-admin && bpm create lms-teacher && bpm create lms-student
                            
                        

                    

                    

                        
🔄

                        
CI/CD Pipeline

                        
AI agent runs E2E tests on staging with pre-authenticated profiles. No manual login needed in CI environment.

                        

                            bpm use staging-qa --headless
                            
                        

                    

                    

                        
🕵️

                        
Web Scraping with Auth

                        
Access authenticated dashboards, admin panels, or member-only content without handling complex OAuth flows.

                        

                            bpm use dashboard-monitor
                            
                        

                    

                    

                        
📊

                        
Data Entry Automation

                        
AI fills forms, uploads files, and submits data across multiple platforms using persistent login sessions.

                        

                            bpm use erp-data-entry
                            
                        

                    

                

            


            
            

                
Example Prompts

                
Copy these prompts to use with your AI agent after configuring MCP.

                

                    

                        

                            Playwright MCP
                            E2E Testing
                        

                        
Navigate to the admin dashboard, create a new user with role "Teacher", verify the success message appears, then check the user list to confirm the new user is listed.

                        
                    

                    

                        

                            Playwright MCP
                            Form Automation
                        

                        
Go to Settings → Profile, update the display name to "Test User", change the timezone to UTC+7, upload avatar.png as profile picture, and click Save.

                        
                    

                    

                        

                            DevTools MCP
                            Debugging
                        

                        
Open the Network tab, navigate to /api/users, find any failed requests (4xx/5xx), and show me the request headers and response body for debugging.

                        
                    

                    

                        

                            Browser Use
                            Natural Language
                        

                        
Log into the LMS as teacher, go to "My Courses", open the first course, check how many students are enrolled, and export the student list as CSV.
